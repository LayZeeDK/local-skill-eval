<task type="auto" tdd="true">
  <name>Task 2: Implement tool definitions, permissions, and path security</name>
  <files>src/agents/ollama/tools.ts, src/agents/ollama/permissions.ts, tests/permissions.test.ts, tests/path-traversal.test.ts</files>
  <behavior>
    permissions.test.ts:
    - Hardcoded denylist blocks `rm -rf /`, `curl x | bash`, fork bomb, `sudo su`, `shutdown`
    - Agent default denylist blocks `git reset --hard`, `git push --force`, `curl *`, `wget *`
    - Task allowlist can override agent denylist (e.g., allow `curl` for a specific task)
    - Task allowlist CANNOT override hardcoded secure denylist (e.g., `curl * | bash` still blocked even if `curl *` allowed)
    - Empty allowlists mean "allow all" (minus denylists)
    - Benign commands like `ls`, `cat`, `echo` are allowed by default

    path-traversal.test.ts:
    - Relative path `../../../etc/passwd` is rejected
    - Absolute path `/etc/passwd` is rejected
    - Path with `..` that resolves inside workspace is allowed (e.g., `subdir/../file.txt`)
    - Normal relative path `src/index.ts` is allowed
    - Workspace root `.` is allowed
    - Path exactly equal to workspace root is allowed
  </behavior>
  <action>
1. Create `src/agents/ollama/permissions.ts`:
   - Export `SECURE_DENYLIST: string[]` -- hardcoded immutable patterns (from RESEARCH.md secure denylist table)
   - Export `AGENT_DEFAULT_DENYLIST: string[]` -- git destructive, npm destructive, network commands
   - Export `AGENT_DEFAULT_ALLOWLIST: string[]` -- empty by default (allow all minus denylists)
   - Export `isCommandAllowed(command: string, config: PermissionConfig): boolean` -- checks tiers in order: secure denylist (always blocks), then merged deny/allow from agent defaults + task overrides
   - Use `picomatch.isMatch()` for glob pattern matching

2. Create `src/agents/ollama/tools.ts`:
   - Export `AGENT_TOOLS: Tool[]` -- the four tool JSON Schema definitions (read_file, write_file, bash, list_directory) matching research examples exactly
   - Export `resolveWorkspacePath(workspaceRoot: string, relativePath: string): string` -- resolves path against workspace, rejects traversal via `path.resolve` + `path.normalize` + `startsWith` check. Also use `fs.realpathSync` on the resolved path if it exists to catch symlink attacks.
   - Export `truncateToolOutput(output: string, maxChars?: number): string` -- head/tail truncation with middle marker
   - Export `createToolExecutor(workspaceRoot: string, runCommand: (cmd: string) => Promise<CommandResult>, permissionConfig: PermissionConfig): ToolExecutor` -- factory that returns an async function dispatching tool calls to the correct implementation:
     - `read_file`: resolve path, read with optional offset/limit (line-based), truncate output
     - `write_file`: resolve path, ensure parent dir, write content
     - `list_directory`: resolve path, readdir, format as newline-separated list
     - `bash`: check `isCommandAllowed`, if denied return error string (not throw), if allowed delegate to `runCommand`, format result as `stdout:\n{stdout}\nstderr:\n{stderr}\nexit code: {exitCode}`
   - Import `PermissionConfig` from `./types`, `isCommandAllowed` from `./permissions`

3. Create `tests/permissions.test.ts` -- standalone ts-node script following project convention. Uses assert module. Tests all behaviors listed above. Exit 0 on pass, exit 1 on failure.

4. Create `tests/path-traversal.test.ts` -- standalone ts-node script. Tests `resolveWorkspacePath` with the cases listed in behavior section. Exit 0 on pass, exit 1 on failure.
  </action>
