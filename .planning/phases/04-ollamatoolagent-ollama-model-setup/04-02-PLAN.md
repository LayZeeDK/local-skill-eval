<task type="auto">
  <name>Task 1: Implement OllamaToolAgent class and smoke test</name>
  <files>src/agents/ollama/index.ts, src/agents/ollama/smoke-test.ts</files>
  <action>
1. Create `src/agents/ollama/index.ts` -- the `OllamaToolAgent` class:
   - Extends `BaseAgent` from `../../types`
   - Constructor accepts `OllamaAgentConfig` (with defaults: model `'qwen3-agent'`, host `'http://localhost:11434'`, maxIterations `30`, maxToolOutputChars `8000`)
   - Creates `Ollama` client instance from `ollama` npm package
   - `async run(instruction, workspacePath, runCommand)`:
     a. Build permission config: load task.toml `[agent.permissions]` if it exists (try/catch, default to empty arrays), merge with `SECURE_DENYLIST` and `AGENT_DEFAULT_DENYLIST`/`AGENT_DEFAULT_ALLOWLIST`
     b. Create tool executor via `createToolExecutor(workspacePath, runCommand, permissionConfig)`
     c. Build system prompt: "You are an AI agent that completes coding tasks. You have access to tools for reading files, writing files, listing directories, and running bash commands. Use these tools to complete the task. When you are done, respond with a summary of what you did."
     d. Initialize messages array: `[{ role: 'system', content: systemPrompt }, { role: 'user', content: instruction }]`
     e. Tool-calling loop (up to maxIterations):
        - Call `ollama.chat({ model, messages, tools: AGENT_TOOLS, stream: false, think: false })`
        - If no `response.message.tool_calls` or empty array: break, return `response.message.content`
        - Push assistant message to messages
        - For each tool call: execute via tool executor, push `{ role: 'tool', content: result }` to messages
     f. After loop (normal exit or max iterations): unload model via `ollama.chat({ model, messages: [], keep_alive: 0 })`
     g. Return final content or `'[Agent reached max iterations without completing]'`
   - Export `OllamaToolAgent`

   Important implementation notes per CONTEXT.md and RESEARCH.md:
   - Use `think: false` in every chat call (disable Qwen3 thinking mode per Pitfall 2)
   - Do NOT parse tool calls from text content -- only use structured `tool_calls` array
   - The `runCommand` callback from the provider already executes in the sandbox -- bash tool delegates to it
   - Wrap the unload in try/catch so agent errors don't prevent unload

2. Create `src/agents/ollama/smoke-test.ts`:
   - Export `async smokeTestToolCalling(host: string, model: string): Promise<{ passed: boolean; error?: string }>`
   - Creates Ollama client, sends a single message "List the files in the current directory." with only the `list_directory` tool definition
   - Uses `think: false` and `options: { num_ctx: 4096, num_predict: 256 }` (small context for speed)
   - Checks `response.message.tool_calls` for a `list_directory` call
   - If tool_calls present with correct name: `{ passed: true }`
   - If no tool_calls but content contains tool-like JSON: `{ passed: false, error: 'Model emits tool calls as text...' }`
   - If no tool_calls and no tool-like content: `{ passed: false, error: 'No tool calls in response...' }`
   - Catch block for connection/timeout errors
  </action>
