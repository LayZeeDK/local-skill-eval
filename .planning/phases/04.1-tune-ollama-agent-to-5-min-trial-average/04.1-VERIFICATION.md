## Warnings (should fix)

### 3. [Scope Sanity] Plan 02 Task 1 is overloaded -- 7 experiments plus apply winning config

- **Plan:** 04.1-02, **Task:** 1
- **Dimension:** scope_sanity
- **Risk:** 28+ eval runs minimum, potentially hours. If an unexpected result mid-sequence leaves a temp Modelfile in place, the executor may apply the wrong combination to production files.
- **Fix:** Split into two tasks: Task 1a runs all experiments and writes experiments.md (auto), Task 1b applies the winning combination and runs final benchmark (auto).

### 4. [Task Completeness] Per-experiment git checkout reverts are fragile for autonomous execution

- **Plan:** 04.1-02, **Task:** 1
- **Dimension:** task_completeness
- **Details:** The action for experiments 2 (structured prompt) and 4 (maxIterations) says to use git checkout to revert source changes after each experiment. For an autonomous agent mid-task, git checkout discards ALL edits to that file, including changes made earlier in the same task. This loses confirmed improvements.
- **Fix:** Remove per-experiment revert instructions. Run all experiments using Modelfile-only changes where possible. Apply ALL KEPT source code changes together at the end of the task.

### 5. [Scope Sanity] Plan 03 Task 1 combines a code change with multiple model benchmarks

- **Plan:** 04.1-03, **Task:** 1
- **Dimension:** scope_sanity
- **Details:** Task 1 implements context pruning (code change in OllamaToolAgent), benchmarks qwen3:4b q4_0, benchmarks qwen3.5:4b (with Ollama version check + smoke test), benchmarks qwen2.5:7b (with thinking suppression changes), applies the best config, and writes escalation.md. A compile error in pruneHistory blocks all model benchmarks.
- **Fix:** Split into Task 1a (implement context pruning + benchmark it) and Task 1b (alternative model benchmarks + apply best config). This also enforces the locked decision to optimize qwen3:4b first.
