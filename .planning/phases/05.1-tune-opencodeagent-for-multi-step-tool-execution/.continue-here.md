<context>
The grader memory contention was the last blocker for consistent reward >= 0.85.
Root cause was two-fold: (1) Ollama keep_alive:0 is async -- model stays resident
briefly after API returns, (2) num_ctx 8192 KV cache (612 MiB) makes CPU inference
too slow even when memory is available. With both fixes, all combos score 1.00.

Key insight: on CPU inference, num_ctx affects speed not just memory. Larger KV cache
means more attention computation per token, even when actual prompt is short.

qwen3:4b scored reward 1.00 (95.3s) with OLLAMA_GRADER_NUM_CTX=2048 in earlier test.
With the default now at 2048, it should work without env var -- needs confirmation.
</context>
