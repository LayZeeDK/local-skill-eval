**2. Rewrite `.github/actions/setup-ollama/action.yml`:**

Replace the `model` input with a `models` input (YAML list of mappings). Keep the `ollama-version` input unchanged.

Input schema for `models`:
```yaml
models:
  description: 'YAML list of model entries. Each entry: name (required), modelfile (optional), as (optional)'
  required: true
```

No default value for `models` -- each caller specifies all needed models explicitly (per user decision).

In the "Start Ollama with optimized config" step, add `OLLAMA_MAX_LOADED_MODELS=1` to both the `GITHUB_ENV` exports and the inline env vars for `ollama serve`.

Replace the single "Pull model" step with a "Pull and create models" step that:
1. Writes `${{ inputs.models }}` to a temp file to avoid subshell variable scoping issues with piped `while read`
2. Parses entries line-by-line using `case` matching on `- name:`, `name:`, `modelfile:`, `as:`
3. For each entry: runs `ollama pull "$current_name"`
4. If entry has both `modelfile` and `as`: runs `ollama create "$current_as" -f "$current_modelfile"`
5. Calls `process_entry` on the last accumulated entry after the loop ends

Use the parsing approach from 06-RESEARCH.md "setup-ollama Model Loop" section, specifically the temp-file variant (NOT the piped version) to avoid subshell variable scoping.

Replace the cache key. The old key was `ollama-model-${{ inputs.model }}`. The new key must hash the full `models` input. Compute the hash in a preceding step:
```yaml
- name: Compute models hash
  id: models-hash
  shell: bash
  run: |
    HASH=$(echo '${{ inputs.models }}' | sha256sum | cut -c1-16)
    echo "hash=$HASH" >> "$GITHUB_OUTPUT"
```
Then use `key: ollama-models-${{ steps.models-hash.outputs.hash }}` in the cache step.

Every `run:` step MUST have explicit `shell: bash` (existing pattern).
